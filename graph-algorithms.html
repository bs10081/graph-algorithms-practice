📋 演算法遍歷方式總整理

演算法 遍歷方式 資料結構 原因
Dijkstra BFS-like Priority Queue（Min-Heap） 每步選 dist 最小的未訪問節點擴展，像 BFS 加上權重排序。邊權都為 1 時行為等同 BFS。
Bellman-Ford BFS-like Array（逐輪遍歷所有邊） 每一輪對所有邊鬆弛，類似 BFS 的層層擴散。第 k 輪後保證找到長度 ≤ k 的最佳路徑。
Floyd-Warshall DP 二維矩陣（三層迴圈） 不是 traversal，是 DP：逐步將節點 k 納入可用中繼站。跟 BFS/DFS 遍歷順序無關。
Prim BFS-like Priority Queue（Min-Heap） 跟 Dijkstra 結構幾乎一樣，從起點透過 Priority Queue 選最小邊向外生長。屬於貪心擴散。
Kruskal 其他 排序陣列 + Union-Find 先把所有邊排序，再依序嘗試加入。不依賴圖的遍歷順序，只看權重和集合關係。
🔑 重點：Dijkstra 是 BFS 在加權圖上的推廣；Prim 跟 Dijkstra 共用同一個核心邏輯（Priority Queue + 貪心選最小）；Bellman-Ford 層層擴散本質也是 BFS；Floyd-Warshall 是 DP；Kruskal 是排序 + Union-Find。
